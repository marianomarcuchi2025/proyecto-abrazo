# Cómo participar sin abrir un issue de GitHub

Este proyecto necesita sobre todo aportes de gente que **no programa** — profesionales clínicos, expertos en accesibilidad, familias. Pedirles que "abran un issue" es más fricción de la necesaria para alguien que solo quiere comentar algo.

**Usá [GitHub Discussions](../../discussions)** para:

- Preguntas ("¿esto sirve para un niño de 4 años?", "¿cómo lo instalo si no sé de computadoras?").
- Comentarios sueltos sobre el flujo de emergencia o el lenguaje, sin necesidad de formatear un reporte de bug.
- Ideas que no son todavía un pedido concreto de cambio.

Reservamos los **issues** para pedidos concretos y accionables (bugs, features específicas, revisión clínica).

## Por qué no hay Discord/Matrix todavía

Cboard e ifme (proyectos comparables, ver `INVESTIGACION_REPOS_POPULARES_Y_ROADMAP.md`) usan Discord y Open Collective. Elegimos no copiar eso todavía: un canal de chat en vivo necesita a alguien moderando y respondiendo en tiempo real, y este proyecto hoy tiene un solo mantenedor. GitHub Discussions no agrega infraestructura nueva, no requiere crear una cuenta en otro servicio, y es asíncrono — se puede responder cuando hay tiempo, sin la presión de un chat en vivo. Si el proyecto crece lo suficiente como para justificar moderación activa, se reevalúa.
